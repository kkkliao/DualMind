📖 说明文档 / Documentation

DualMind 是什么

DualMind 是一个本地双 AI 群组协作界面，把用户、OpenClaw 和 Hermes 放在同一个群聊里。日常聊天两位 AI 都可以发言；当前主 AI 始终是站位执行者，真正涉及文件、命令或配置时只允许主 AI 操作，副 AI 负责沟通、校准、辩论、复核和提醒风险。

🚀 基本使用

  1. 先在设置页检测 OpenClaw 和 Hermes 是否可用。
  2. 选择两个角色模式之一：OpenClaw 主执行或 Hermes 主执行。
  3. 在群聊输入消息；不点名时按当前主副关系自然协作。
  4. 使用 @OpenClaw 或 @Hermes 只会改变谁先回复，不会改变当前执行者。
  5. 涉及代码、命令、文件或配置修改时，系统会创建执行权租约，只让当前主 AI 操作。

🛡️ 执行与安全

DualMind 默认把聊天和执行分开：聊天可以自由讨论，执行必须加锁。副 AI 可以反驳、纠正、建议、阻断风险，但不能声称自己改了文件、运行了命令，也不能说等拿到执行权后再写入或保存。危险操作需要用户确认；微信等远程消息默认只能进入回合记录，不能直接触发代码执行。

🌊 输出与回合记录

AI 回复前会显示思考动画。当前 OpenClaw/Hermes CLI 适配器使用模拟流式输出：服务端拿到完整回复后再分块显示。每一轮都会写入结构化回合记录，可查看参与者、执行者、Action Lease、状态、消息和事件。

📱 微信集成

DualMind 内置微信消息桥接入口，支持把远程消息接入群组。发布公开版本时建议保持远程代码执行关闭。

  1. 打开设置面板，勾选“启用微信”
  2. 点击“📱 获取二维码”
  3. 复制链接在浏览器打开，用微信扫码
  4. 在微信中确认授权
  5. 点击“✅ 已扫码，开始配对”
  6. 配对后微信消息进入回合记录，由用户在 Web UI 继续处理

🧭 群聊校准

如果一个 AI 的回答出现明显事实、时间、角色或安全问题，另一个 AI 可以主动纠正；被纠正方会简短确认理解并收口。没有问题时不会额外追加无意义消息。

❓ 常见问题

Q: OpenClaw 不回复？

检查 Gateway 是否运行。终端运行 curl localhost:18789/health 验证。

Q: Hermes 不回复？

确认 Hermes 已安装。hermes --version

Q: 微信消息收不到？

重新配对微信插件。

Q: 怎么改端口？

设置 → 端口 → 保存 → 重启服务。
